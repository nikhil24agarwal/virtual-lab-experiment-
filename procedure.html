Routing Protocol

Static routing protocol:

  1. When the user enters this page, they will firstly see the option of a new graph, clicking on it will generate a graph on random basis from the already fed 3 graphs in the system containing 5 nodes.
  2. As soon as the new graph is added, the tables below the graph (one table for each node) will automatically get updated depicting the connected nodes.
  3. If the user wishes to add an extra edge they can do so by specifying the from and to destination along with the weight to connect the two nodes.
  4. After the new edges have been added, the table will once again get updated accordingly.
  5. Lastly the user has to end the information for sending the data packet, i.e. by specifying the source node and destination node in the space provided on the right hand side of our webpage.
  6. The best path will be chosen and displayed to the user using red lines.

RIP protocol:

  1. When the user enters this page, they will firstly see the option of a new graph, clicking on it will generate a graph on random basis from the already fed 3 graphs in the system containing 5 nodes.
  2. The initial table will be created automatically and the table for each node will be updated (following the Bellman Ford Algorithm) by clicking on the update button, provided on the top right corner.
  3. If the user wishes to add an extra edge they can do so by specifying the from and to destination along with the weight to connect the two nodes.
  4. After the new edges have been added, the table will once again get updated accordingly by clicking on the update button once again.
  5. Lastly the user has to end the information for sending the data packet, i.e. by specifying the source node and destination node in the space provided on the right hand side of our webpage.
  6. The best path will be chosen and displayed to the user using red lines.
